# dev-log — 앱 개발 이슈·학습 문서화

앱 개발 중 만난 버그·에러·디버깅 과정·조치 내역을 **프로젝트별로** 정리해 모아두는 곳.
문서는 두 종류로 나뉜다.

- **이슈 문서** — 버그/에러를 겪고 고친 *사건* 기록 → 프로젝트 폴더에 날짜별 (`<프로젝트>/YYYY-MM-DD-*.md`)
- **학습 노트** — 그 과정에서 건진 재사용 가능한 *개념* → 공용 폴더에 주제별, 파일명 앞 생성일자 (`learn/YYYY-MM-DD-<주제>.md`)

## 사용법

어느 프로젝트에서든 Claude Code에서:

```
/dev-log
```

호출 후 에러 메시지 / 스택 트레이스 / 빌드 로그 / 콘솔 출력 / 코드 조각 / "이렇게 고쳤다" 메모,
또는 "이 개념 학습 노트로 정리해줘"를 붙여 넣으면 된다. 결과는 아래 구조로 자동 저장되고,
**이 저장소(`~/docs/dev-log`)에 자동으로 커밋·푸시된다.** (`main` → `origin/main`)

흐름: 문서 생성 → `~/docs/dev-log/`에 저장 + 월별 README 인덱스에 한 줄 추가 → `git add -A && git commit && git push` 자동.
푸시가 실패하면 커밋만 로컬에 남고 에러를 알려준다 → 나중에 `git -C ~/docs/dev-log push`.

## 폴더 구조

```
~/docs/dev-log/
├─ <프로젝트>/                                  ← 이슈 정본만(날짜별). git 리포 폴더명으로 자동
│  └─ 2026-06-02-cordova-camera-permission.md
├─ learn/                                       ← 학습 노트(주제별 + 생성일자, 프로젝트 무관)
│  ├─ 2026-06-02-promise-error-handling.md
│  └─ cordova/2026-06-02-plugin-lifecycle.md    ← 큰 주제는 하위 폴더
└─ months/                                      ← 월별 시간순 뷰 (마크다운 README 인덱스)
   └─ 2026-06/
      └─ README.md   ← 이 달 이슈를 시간순으로 모은 마크다운 링크 표 (정본으로 가는 상대경로 링크)
```

- **이슈**: 프로젝트 폴더에 날짜별로(=정본). "이 앱 기록"이 한곳에 모임.
- **학습**: 프로젝트 밖 공용 `learn/`에 주제별로. 나중에 주제로 검색.
- **months/**: 각 이슈를 월별 `README.md` **마크다운 인덱스**에 한 줄씩 추가해, 프로젝트를 가로질러 시간순으로 훑기 가능.
  (정본은 프로젝트 폴더에 그대로 있고, 월 폴더 README엔 정본으로 가는 상대경로 마크다운 링크만. GitHub에서 클릭 가능하게 렌더됨.)

## 스킬 본체 위치

- `~/.claude/skills/dev-log/SKILL.md` ← 실제 프롬프트(스킬). 계정 전체에서 `/dev-log` 로 호출.
- 이 README는 사본/안내.

## 핵심 원칙 (요약)

1. 입력에 없는 사실은 지어내지 않는다.
2. **불확실하면 표시만 하고 넘어가지 말고 — 먼저 부족한 정보를 되묻거나 확인 지점을 제안해 채운다.**
   최종 문서엔 "확실하지 않음" 미결 표식을 안 남기는 게 목표. (끝내 확인 불가할 때만 `🔎 추가 조사 필요`)
3. 증상 / 원인 / 조치 / 결과를 분리한다.
4. 원인을 계층으로 분류: 클라이언트 / 빌드·배포 / 네이티브·플러그인 / API·네트워크 / 데이터 / 디바이스·OS.
5. 재현 절차·환경(OS·기기·앱 버전·브랜치/커밋)을 기록한다.
6. 재발 방지 + 트러블슈팅 가이드까지 채운다. 한국어 작성, 코드·에러 원문은 원문 유지.
